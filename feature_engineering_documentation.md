# Feature Engineering Documentation

**Project:** Customer Churn Prediction
**Dataset:** customer_churn.csv

---

# 1. Introduction

Feature engineering is the process of creating new variables (features) from existing data to improve machine learning model performance. Well-engineered features help models capture hidden patterns that may not be visible in the raw dataset.

In this project, several new features were created using combinations and transformations of existing variables to improve the predictive ability of the churn prediction model.

---

# 2. Objectives of Feature Engineering

The main goals of feature engineering in this project were:

* Improve predictive power of the dataset
* Capture hidden relationships between variables
* Represent customer behavior more effectively
* Create business-meaningful indicators of churn risk

A total of **6 engineered features** were created from the original dataset.

---

# 4. Feature Descriptions

## 4.1 Average Monthly Spend

**Feature Name:** `AvgMonthlySpend`

This feature estimates how much a customer spends per month based on their total charges and tenure.

### Formula

```text
AvgMonthlySpend = TotalCharges / (Tenure + 1)
```

The value `+1` prevents division by zero when tenure is zero.

### Implementation

```python
df["AvgMonthlySpend"] = df["TotalCharges"] / (df["Tenure"] + 1)
```

### Business Insight

Customers with consistently higher spending patterns may exhibit different churn behavior compared to low-spending customers.

---

## 4.2 Customer Lifetime Value (CLV)

**Feature Name:** `CLV`

Customer Lifetime Value estimates the overall value generated by a customer over their subscription duration.

### Formula

```text
CLV = MonthlyCharges × Tenure
```

### Implementation

```python
df["CLV"] = df["MonthlyCharges"] * df["Tenure"]
```

### Business Insight

Customers with high lifetime value are often prioritized for retention strategies.

---

## 4.3 Payment Efficiency

**Feature Name:** `PaymentEfficiency`

This feature evaluates the relationship between the customer's total payments and monthly charges.

### Formula

```text
PaymentEfficiency = TotalCharges / (MonthlyCharges + 1)
```

### Implementation

```python
df["PaymentEfficiency"] = df["TotalCharges"] / (df["MonthlyCharges"] + 1)
```

### Business Insight

This feature helps identify irregular payment behavior that might correlate with churn.

---

## 4.4 Long-Term Customer Indicator

**Feature Name:** `LongTermCustomer`

This binary feature identifies whether a customer has remained with the company for a long duration.

### Condition

```text
LongTermCustomer = 1 if Tenure > 24 months else 0
```

### Implementation

```python
df["LongTermCustomer"] = (df["Tenure"] > 24).astype(int)
```

### Business Insight

Customers who stay longer typically have a lower probability of churn.

---

## 4.5 High Spender Indicator

**Feature Name:** `HighSpender`

This binary feature identifies customers whose monthly charges exceed the dataset average.

### Condition

```text
HighSpender = 1 if MonthlyCharges > Average MonthlyCharges
```

### Implementation

```python
df["HighSpender"] = (df["MonthlyCharges"] > df["MonthlyCharges"].mean()).astype(int)
```

### Business Insight

High-spending customers may require special retention strategies.

---

## 4.6 Charges per Tenure

**Feature Name:** `ChargesPerTenure`

This feature represents the average charges relative to the length of the customer relationship.

### Formula

```text
ChargesPerTenure = TotalCharges / (Tenure + 1)
```

### Implementation

```python
df["ChargesPerTenure"] = df["TotalCharges"] / (df["Tenure"] + 1)
```

### Business Insight

This feature helps normalize spending behavior across customers with different subscription durations.
